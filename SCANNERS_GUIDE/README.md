# SCANNERS GUIDE — сканеры QR/штрих-кодов камерой телефона

Папка содержит **готовую, законченную реализацию 4 разных сканеров QR/штрих-кодов** для веб-страниц, работающих на телефоне через браузер. Всё извлечено из рабочего проекта `phone-scan-main` (PWA "Приёмка ткани").

## Какие сканеры реализованы

1. **html5-qrcode** (2.3.8) — самый простой и надёжный, главный выбор по умолчанию. Автоматически рисует UI сканера внутри заданного `<div>`.
2. **jsQR** (1.4.0) — лёгкая библиотека, только QR-коды. Ручной цикл через `<video>` + `<canvas>` + `requestAnimationFrame`. Используется дважды — как «ZBar» и как отдельный «jsQR» (в проекте это fallback-ы).
3. **Scanbot Web SDK** (8.0.1) — профессиональная коммерческая библиотека. Поддерживает десятки форматов (QR, EAN, DataMatrix, PDF417, Code128 и т.д.), работает отлично в плохом свете и под углом. **Бесплатно работает 60 секунд без лицензионного ключа**, после чего требуется триал/подписка.
4. **Ручной ввод** — текстовое поле на случай, если код плохо читается камерой.

## Формат кода в проекте-доноре

В исходном проекте код имеет вид `"партия_рулон"` — например `2020_1`. Функция `handleCodeScan()` парсит по `_` и ищет партию. **В вашем новом проекте эту функцию надо заменить на свою логику** — место пометки специально изолировано.

## Требования для работы камеры

- **HTTPS обязателен** (кроме `localhost`). Без HTTPS браузер не даст доступ к камере. В проекте используется Tailscale Funnel → см. `tailscale-*.bat`.
- **Разрешение на камеру** пользователь даёт при первом запуске (запрос браузера).
- **Задняя камера** (`facingMode: 'environment'`) — включена по умолчанию во всех сканерах.

## Как пользоваться этой папкой

**Человеку:** откройте `06-minimal-example.html` в браузере (по HTTPS или с `localhost`) — всё работает из коробки, кроме Scanbot (ему нужны локальные WASM-файлы — см. `05-scanbot-detailed.md`).

**AI-ассистенту:** дайте ему `PROMPT.md` вместе с этой папкой. Он воспроизведёт функционал 1-в-1.
